Struct poli_math::Matrix4

source · 
#[repr(C)]
pub struct Matrix4 {
    pub elements: [f32; 16],
}
Expand description

4x4 matrix, commonly used to encode transformations i.e. translation, rotation, and scale. Note that elements are stored in column-major order due to the fact that WGSL uses column-major ordering.

Fields§

§elements: [f32; 16]

The elements of this matrix in column-major order.

pub fn compose( translation: &Vector3, rotation: &Quaternion, scale: &Vector3, ) -> Self

Creates a matrix for the transformation composed of the given translation, rotation, and scale. This uses TRS ordering: scale first, then rotation, then translation.

The implementation is based on the formulae for quaternion to 4x4 matrix conversion, with the addition of scale and translation.
